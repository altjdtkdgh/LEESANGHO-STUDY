**filter(필터):** 필터는 DispatcherServlet 이전에 실행되는데 요청내용을 변경하거나 여러가지 체크를 수행. 보통 web.xml에 등록하고, 일반적으로 인코딩 변환 처리, XSS 방어 등의 요청에 대한 처리 사용.

**interceptor(인터셉터):** DispatcherServlet이 컨트롤러를 호출하기 전, 후로 끼어들기 때문에 스프링 컨텍스트 영역 내부에서 Controler에 관한 요청과 응답에 대해 처리.( 스프링 모든 빈 객체 접근 가능) 로그인 체크, 권한체크, 로그 확인 업무처리.

**AOP:** 위와 달리 메소드 전후의 지점에 자유롭게 설정 가능. 인터셉터와 필터는 주소로 대상을 구분해서 걸러내야하는 반면, AOP는 주소,파라미터, 애노테이션등 다양한 방법으로 대상을 지정할수 있음. 권한, 로깅, 트랜잭션등 처리.
